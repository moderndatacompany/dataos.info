##

While creating Lens on Postgres Depot the following aspects need to be considered:

* The SQL dialect used in the `model/sql` folder to load data from the Postgres source should be of the Postgres dialect.

* The table naming in the `model/table`  should be of the format: `schema.table`.

## Depot manifest file

```
name: postgres
version: v2alpha
type: depot
layer: user
depot:
  type: jdbc                 
  description: default postgres depot
  external: true
  secrets:
    - name: postgres-rw
      keys: 
        - postgres-rw
      allkeys: true

    - name: postgres-r
      keys: 
        - postgres-r
      allkeys: true
    
  jdbc:
    subprotocol: postgresql
    host: usr-db-dataos-ck-oynn-dataostra-stg.postgres.database.azure.com
    port: 5432
    database: postgres
```

## Deployment manifest file

The below manifest is intended for source-type depot named `postgresdepot`, created on the Postgres source.

```yaml
version: v1alpha
name: "postgres-lens"
layer: user
type: lens
tags:
  - lens
description: postgres depot lens deployment on lens2
lens:
  compute: runnable-default
  secrets:
    - name: bitbucket-cred
      allKeys: true
  source:
    type: depot #minerva/themis/depot
    name: postgresdepot
  repo:
    url: https://bitbucket.org/tmdc/sample
    lensBaseDir: sample/lens/source/depot/postgres/model 
    # secretId: lens2_bitbucket_r
    syncFlags:
      - --ref=main #repo-name

  api:   # optional
    replicas: 1 # optional
    logLevel: info  # optional
      
    resources: # optional
      requests:
        cpu: 100m
        memory: 256Mi
      limits:
        cpu: 2000m
        memory: 2048Mi

    logLevel: debug  # optional

        memory: 256Mi
```
