    Configuration Fields¶

    flare¶

    Description: Flare Stack specific section.
    Data Type: object
    Requirement: mandatory
    Default Value: none
    Possible Value: none
    Example Usage: 

    flare: {}

    driver¶

    Description: the driver is the controller node of the operation and controls the operations of the workflow.
    Data Type: object
    Requirement: optional
    Default Value: none
    Possible Value: none
    Example Usage: 

    driver:
  coreLimit: 1000m
  cores: 1
  memory: 1024m

    executor¶

    Description: the executor is the worker node that performs the operations defined in the workflow and sends the result back to the driver. It must interface with the cluster manager in order to actually get physical resources and launch executors.
    Data Type: object
    Requirement: optional
    Default Value: none
    Possible Value: none
    Example Usage: 

    executor:
  coreLimit: 2000m
  cores: 2
  instances: 2
  memory: 1024m

    job¶

    Description: jobs can be defined as the steps to complete any action such as loading initial data, updating data, or writing data to sink.
    Data Type: object
    Requirement: mandatory
    Default Value: none
    Possible Value: none
    Example Usage: 

    job: {}

    explain¶

    Description: use this flag to print the spark logical/physical plans in the logs.
    Data Type: boolean
    Requirement: optional
    Default Value: false
    Possible Value: true/false
    Example Usage: 

    explain: true

    logLevel¶

    Description: a log level is a piece of information from a given log message that distinguishes log events from each other. It helps in finding runtime issues with your Flare Stack and in troubleshooting.
    Data Type: string
    Requirement: optional
    Default Value: INFO
    Possible Value: INFO / WARN / DEBUG/ ERROR
    Example Usage: 

    logLevel: INFO

    streaming¶

    Description: the streaming section contains key-value properties related to executing the stream jobs.
    Data Type: object
    Requirement: optional
    Default Value: none
    Possible Value: none
    Additional Information: more information regarding the streaming section can be found here
    Example Usage: 

    streaming: {}

    inputs¶

    Description: contains dataset details for reading data from supported data sources.
    Data Type: object
    Requirement: mandatory
    Default Value: none
    Possible Value: none
    Additional Information: more information regarding the inputs section can be found here
    Example Usage: 

    inputs: {}

    steps¶

    Description: one or more sequences to execute the steps defined for performing any transformation or applying any flare function or command.
    Data Type: object
    Requirement: mandatory
    Default Value: none
    Possible Value: none
    Additional Information: more information regarding the steps section can be found here
    Example Usage: 

    steps: {}

    outputs¶

    Description: contains dataset details for writing data to supported data sources.
    Data Type: object
    Requirement: mandatory
    Default Value: none
    Possible Value: none
    Additional Information: more information regarding the outputs section can be found here
    Example Usage: 

    outputs: {}

    assertions¶

    Description: assertions allow you to perform validation checks on top of pre-written datasets. They are defined under the assertions section.
    Data Type: object
    Requirement: optional
    Default Value: none
    Possible Value: none
    Additional Information: more information regarding the assertions section can be found here
    Example Usage: 

    assertions: {}

    actions¶

    Description: Iceberg-specific actions are defined in the actions section.
    Data Type: object
    Requirement: optional
    Default Value: none
    Possible Value: none
    Additional Information: more information regarding the actions section can be found here
    Example Usage: 

    actions: {}
