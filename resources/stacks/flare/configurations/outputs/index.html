    Flare Output Section Grammar¶

    The Flare Output Section YAML Configuration provides users with the necessary tools to define the output dataset settings for writing data to different destinations. It enables users to specify the desired format, compression, partitioning, and other options for the output dataset.

    Structure of the Outputs Section¶

    Common Configuration¶

    outputs:
  - name: {{account_connect}}
    dataset: {{dataos://gcdexport:none/gcdcore_account?acl=rw}}
    format: {{csv}}
    Flare Output Section Common YAML Configuration

    Advanced Configuration¶

    outputs:
  - name: {{top_100_accounts}}
    dataset: {{dataos://icebase:bronze/topaccounts?acl=rw}}
    format: {{iceberg}}
    driver: {{org.apache.jdbc.psql.Driver}}
    title: {{Account}}
    description: {{Account data from GCD export}}
    tags:
      - {{Lookup-Tables}}
      - {{Accounts}}
    options:
      saveMode: {{overwrite}}
      extraOptions:
        {{key1: value1}}
      compressionType: {{gzip}}
      sort:
        mode: {{partition}}
        columns:
          - name: {{version}}
            order: {{desc}}
      iceberg:
        merge:
          onClause: {{old.id = new.id}}
          whenClause: {{matched then update set * when not matched then insert *}}
        properties:
          write.format.default: {{parquet}}
          write.metadata.compression-codec: {{gzip}}
        partitionSpec:
          - type: {{identity}}
            column: {{version}}
          - type: {{day}}
            column: {{timestamp}}
            asColumn: {{day_partitioned}}
    Flare Output Section Advanced YAML Configuration

    Configuration Fields¶

    outputs¶

    Description: The outputs section comprises of configurations for sinking any of the views created during input load or sequence step into a dataset.
    Data Type: Object
    Requirement: Mandatory
    Default Value: None
    Possible Value: None
    Example Usage:

    outputs:
  {} # Properties within the outputs section

    name¶

    Description: Name assigned to one of the views you want to sink as an output dataset.
    Data Type: String
    Requirement: Mandatory
    Default Value: None
    Possible Value: None
    Example Usage:

    name: top_100_accounts

    dataset¶

    Description: DataOS UDL address specifying the location where the output dataset will be stored. dataset where you want to write your data referred by view in name above.
    Data Type: String
    Requirement: Mandatory
    Default Value: None
    Possible Value: Any valid UDL address except the input dataset address
    Example Usage:

    dataset: dataos://icebase:bronze/topaccounts?acl=rw

    format¶

    Description: File format in which the output dataset will be saved.
    Data Type: String
    Requirement: Optional
    Default Value: Based on output depot type
    Possible Value: iceberg/parquet/json/kafkaavro/kafkajson/pulsar/bigquery
    Example Usage:

    format: iceberg

    driver¶

    Description: The field can be used to override the default driver class in case of output dataset to be stored in JDBC depot types.
    Data Type: String
    Requirement: Optional
    Default Value: None
    Possible Value: None
    Example Usage:

    driver: org.apache.jdbc.psql.Driver

    title¶

    Description: Title of the output dataset.
    Data Type: String
    Requirement: Optional
    Default Value: None
    Possible Value: Any string
    Example Usage:

    title: Account

    description¶

    Description: Description of the output dataset.
    Data Type: String
    Requirement: Optional
    Default Value: None
    Possible Value: Any string
    Example Usage:

    description: Account data from GCD export

    tags¶

    Description: List of tags or labels associated with the output dataset.
    Data Type: List of strings
    Requirement: Optional
    Default Value: None
    Possible Value: Any string
    Example Usage:

    tags:
  - Lookup-Tables
  - Accounts

    options¶

    Description: Additional options or configurations for the output dataset. They are specified as key-value pairs. These include properties for saveMode, partitioning, etc.
    Data Type: Object
    Requirement: Optional
    Default Value: None
    Possible Value:Any option supported by the underlying Spark datasource connector
    Example Usage:

    options:
  saveMode: overwrite
  extraOptions:
    key1: value1
  compressionType: gzip
  sort:
    mode: partition
    columns:
      - name: version
        order: desc

    saveMode¶

    Description: saveMode is used to specify the expected behavior of saving a Dataframe to a data source
    Data Type: String
    Requirement: Optional
    Default Value: overwrite
    Possible Value:overwrite/append
    Example Usage:

    saveMode: overwrite

    extraOptions¶

    Description: extraOptions is used to specify some additional key-value properties
    Data Type: Map(String,String)
    Requirement: Optional
    Default Value: None
    Possible Value:None
    Example Usage:

    extraOptions:
  key1: value1

    compressionType¶

    Description: Type of Compression
    Data Type: String
    Requirement: Optional
    Default Value: gzip
    Possible Value:gzip/snappy
    Example Usage:

    compressionType: gzip

    sort¶

    Description: This field comprises of properties for sorting column values in ascending or descending order.
    Data Type: Object
    Requirement: Optional
    Default Value: None
    Possible Value:None
    Additional Details: In partition mode, chunk of data or logical division of data are stored on a node in the cluster in Iceberg kind of dataset.
    Example Usage:

    sort:
  mode: partition
  columns:
    - name: version # column name
      order: desc # order (asc, desc)

    iceberg¶

    Description: Configuration specific to the Iceberg format. This section allows you to specify additional Iceberg-specific options and properties for the output dataset.
    Data Type: Object
    Requirement: Optional
    Default Value: None
    Possible Value: None
    Example Usage:

    iceberg:
  merge:
    onClause: old.id = new.id
    whenClause: matched then update set * when not matched then insert *
  properties:
    write.format.default: parquet
    write.metadata.compression-codec: gzip
  partitionSpec:
    - type: identity
      column: version
    - type: day
      column: timestamp
      asColumn: day_partitioned

    merge¶

    Description: Configuration for merge operation in Iceberg. Specifies the merge behavior for updates and inserts.
    Data Type: Object
    Requirement: Optional
    Default Value: None
    Possible Value: None
    Example Usage:

    merge:
  onClause: old.id = new.id
  whenClause: matched then update set * when not matched then insert *

    onClause¶

    Description: The ON clause to define the merge condition. Specifies the condition to match rows for update or insert operations.
    Data Type: String
    Requirement: Optional
    Default Value: None
    Possible Value: None Example Usage:

    onClause: old.id = new.id

    whenClause¶

    Description: The WHEN clause to define the merge behavior. Specifies the action to be performed for matched and unmatched rows.
    Data Type: String
    Requirement: Optional
    Default Value: None
    Possible Value: None
    Example Usage:

    whenClause: matched then update set * when not matched then insert *

    properties¶

    Description: Additional properties to be set for the Iceberg writer. Allows you to specify custom properties or override default properties.
    Data Type: Map (String, String)
    Requirement: Optional
    Default Value: None
    Possible Value: None
    Example Usage:

    properties:
  write.format.default: parquet
  write.metadata.compression-codec: gzip

    partitionSpec¶

    Description: Partitioning configuration for the output dataset. Specifies how the data should be partitioned in the Iceberg format.
    Data Type: List of Objects
    Requirement: Optional
    Default Value: None
    Possible Value: None
    Example Usage:

    partitionSpec:
  - type: identity 
    column: version 
  - type: day
    column: timestamp
    asColumn: day_partitioned

    type¶

    Description: The type of partitioning to be applied. Specifies the partitioning strategy.
    Data Type: String
    Requirement: Mandatory
    Default Value: None
    Possible Value: identity/year/month/day/hour
    Example Usage:

    type: identity

    column¶

    Description: The column to be used for partitioning. Specifies the column name on which the partitioning should be based.
    Data Type: String
    Requirement: Optional
    Default Value: None
    Possible Value: None
    Example Usage:

    column: timestamp

    asColumn¶

    Description: Specifies the column name after paritioning.
    Data Type: String
    Requirement: Optional
    Default Value: None
    Possible Value: None
    Example Usage:

    asColumn: day_partitioned
